# TASKS.md — ลำดับงานสร้าง "บีนส์ ERP"

กติกา: ทำทีละ TASK ตามลำดับ · จบแต่ละ TASK ต้องเปิดผลงานให้ดู + ทดสอบเองก่อนรายงาน
· ผู้ใช้จะพิมพ์แค่ "ทำ TASK ถัดไป" — ให้เช็คเองว่าถึงข้อไหนแล้ว

## TASK 1 — Showcase ทั้งระบบ (index.html)
สร้าง mockup โบรชัวร์ 1 หน้า โชว์ 5 โมดูลตาม SPEC (ภาพนิ่ง ข้อมูลตัวอย่างสมจริง ยังไม่ต้องกดได้จริง)
ใช้ Design System ใน CLAUDE.md เคร่งครัด — นี่คือหน้าไว้นำเสนอผู้บริหาร/หุ้นส่วน
✅ เสร็จเมื่อ: เปิดแล้วเลื่อนดูเห็นครบ 5 จอ สวยเป็นระบบเดียวกัน ใช้บนมือถือได้

## TASK 2 — โครงแอพจริง (app.html)
โครง shell ตาม Design System: sidebar ซ้าย (โลโก้ร้าน + เมนู 5 เมนู) + topbar + ตั้งค่า localStorage "beansERP" + seed ข้อมูลตาม SPEC
(เมนูอื่นเป็นหน้าว่างมีชื่อกำกับไว้ก่อน) — Design System เดียวกับ showcase
✅ เสร็จเมื่อ: สลับเมนูได้ครบ รีเฟรชแล้วข้อมูล seed ยังอยู่

## TASK 3 — เมนูขาย (POS)
ตาม SPEC เต็ม: เมนูการ์ด · โต๊ะ · ตะกร้า · บันทึกออเดอร์ · เช็คบิลเงินสด/โอน · ประวัติบิลวันนี้
✅ เสร็จเมื่อ: สั่งอเมริกาโน่ 2 + เค้ก 1 ลงโต๊ะ 3 → เช็คบิลได้ 155 บาท → ประวัติขึ้น · รีเฟรชไม่หาย

## TASK 4 — สต๊อก + BOM (ตัดอัตโนมัติ)
เมนูสต๊อก + เมนูสูตร BOM ตาม SPEC และผูกกับการเช็คบิล: จ่ายเงินปุ๊บ ตัดวัตถุดิบตามสูตรทุกรายการ
✅ เสร็จเมื่อ: ขายลาเต้ 2 แก้วแล้วเช็คบิล → นมสดลด 300ml เมล็ดลด 36g แก้วลด 2 · รับของเข้าแล้วกลับมาเพิ่ม

## TASK 5 — เช็คอินสแกนหน้า (กล้อง)
เมนูเช็คอินตาม SPEC: เลือกชื่อ → กล้องเปิด → ถ่ายหน้า → บันทึกรูป+เวลา เข้า/ออกงาน
สำคัญ: ถ้าเบราว์เซอร์ไม่ให้เปิดกล้อง ต้องเช็คอินแบบไม่มีรูปได้ + ขึ้นข้อความว่ากล้องจะใช้ได้เมื่อขึ้นเว็บจริง (https)
✅ เสร็จเมื่อ: เช็คอินได้ทั้งแบบมีกล้องและไม่มีกล้อง ประวัติวันนี้โชว์เวลา (และรูปถ้ามี)

## TASK 6 — แดชบอร์ดเจ้าของ (หน้าแรก)
ตาม SPEC เต็ม: การ์ดตัวเลขวันนี้ · ขายดี 3 อันดับ · วัตถุดิบเหลือง/แดง · ใครเข้างาน · แถบเตือนบนสุด
ทุกตัวเลขดึงจากข้อมูลจริง ห้ามใส่เลขปลอม
✅ เสร็จเมื่อ: ตัวเลขบนแดชบอร์ดตรงกับที่ขาย/ตัดสต๊อก/เช็คอินจริงทุกตัว

## TASK 7 — ตรวจรวม + ปิดงาน
ไล่ทดสอบทั้งแอพเหมือนผู้ใช้จริง (ขาย→ตัดสต๊อก→เช็คอิน→ดูแดชบอร์ด รวมเคสแปลก ๆ: จำนวน 0, สต๊อกใกล้หมด,
โต๊ะค้างนาน) แก้ทุกอย่างที่เจอ + เช็คว่าทุกตารางมีปุ่ม CSV แล้วรายงานผลการตรวจเป็นข้อ ๆ
✅ เสร็จเมื่อ: รายงานการตรวจระบุว่าลองอะไร เจออะไร แก้อะไรไปแล้วบ้าง

หน้าให้ลูกค้ากดสั่งเอง — **ทำตามหัวข้อ "หน้าตา order.html" ใน SPEC.md เคร่งครัด**:

ชั้นยืนยัน (ชื่อ/เม็ดเลือกโต๊ะ/เบอร์) → หน้าขอบคุณเลขคิวตัวใหญ่ — อารมณ์แอพสั่งกาแฟจริง
ห้ามมีข้อมูลหลังบ้าน (ยอดขาย/สต๊อก/ปุ่มจัดการ) โผล่ในหน้านี้เด็ดขาด

## TASK 9 — เมนูออเดอร์เข้า (หลังบ้านรับออเดอร์)
เมนู 🛎 ใน app.html ตาม SPEC ภาค 2: การ์ดออเดอร์เรียงตามเวลา สีตามสถานะ · ปุ่มเลื่อนสถานะ
ใหม่ → กำลังทำ → เสร็จรอรับ → รับแล้ว+เก็บเงิน (จุดนี้ตัดสต๊อกตาม BOM + นับยอดขาย)
· ออเดอร์ใหม่ต้องโผล่เองภายใน ~3 วิ + badge ออเดอร์ค้างบน header ทุกเมนู
✅ เสร็จเมื่อ: เปิด order.html อีกเมนู สั่งลาเต้ 1 → หลังบ้านเด้งออเดอร์ใหม่เอง → ไล่สถานะจนรับแล้ว
