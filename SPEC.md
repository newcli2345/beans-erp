# SPEC.md — ระบบ "บีนส์ ERP"

## ระบบนี้คืออะไร
ระบบหลังบ้านครบวงจรของร้านบีนส์ในแอพเดียว: ขายหน้าร้าน → ตัดสต๊อกตามสูตรอัตโนมัติ →
พนักงานเช็คอินด้วยกล้อง → เจ้าของเห็นทุกอย่างในแดชบอร์ด

## ไฟล์ของระบบ
- `index.html` = Showcase (โบรชัวร์ mockup ทั้งระบบ ไว้นำเสนอ — ไม่ใช่แอพจริง)
- `app.html` = แอพหลังบ้านโครง shell (sidebar ซ้าย + topbar) เมนู 6 เมนู: 📊 แดชบอร์ด (หน้าแรก) · 🛎 ออเดอร์เข้า · 🛒 ขาย · 📦 สต๊อก · 🧾 สูตร BOM · 📸 เช็คอิน
- `order.html` = หน้าลูกค้า (ภาค 2) — เมนูสวย ๆ ให้ลูกค้ากดสั่งเอง ไม่มีปุ่ม/ข้อมูลหลังบ้านโผล่เด็ดขาด

## ข้อมูลในระบบ (localStorage คีย์ "beansERP")
```
{
  orders:   [{ id, source: "pos"|"online", table, customer:{name, phone}|null,
               items:[{menuId, qty}], total,
               status: "open"|"paid" (pos) หรือ "new"|"cooking"|"ready"|"paid" (online),
               createdAt, paidAt, payMethod }],
  stock:    [{ id, name, unit, qty, initQty }],
  checkins: [{ id, staffId, time, type: "in"|"out", photo }]   // photo = dataURL หรือ null
}
```
ค่าคงที่ในโค้ด (ไม่ต้องแก้ได้จากหน้าจอ):
- MENU 6 รายการพร้อมราคา (ตาม CLAUDE.md)
- STAFF: มายด์ · บอส · แนน
- STOCK เริ่มต้น: เมล็ดกาแฟ 2000 g · นมสด 6000 ml · ผงโกโก้ 500 g · ชาไทย 500 g · แก้ว/บรรจุภัณฑ์ 100 ใบ
- BOM (สูตรต่อ 1 หน่วยขาย):
  อเมริกาโน่ = เมล็ด 18g + แก้ว 1 · ลาเต้ = เมล็ด 18g + นม 150ml + แก้ว 1
  คาปูชิโน่ = เมล็ด 18g + นม 120ml + แก้ว 1 · ชาไทย = ชาไทย 15g + นม 100ml + แก้ว 1
  โกโก้ = ผงโกโก้ 20g + นม 150ml + แก้ว 1 · เค้กกล้วยหอม = แก้ว/บรรจุภัณฑ์ 1

## กติกาการไหลของข้อมูล (หัวใจของระบบ)
1. กด "เช็คบิล" ในเมนูขาย → order เป็น paid → **ตัดสต๊อกตาม BOM ของทุกรายการในบิลทันที**
2. ยอดขาย/จำนวนบิล/เมนูขายดี ในแดชบอร์ด คำนวณจาก orders ที่ paid เท่านั้น
3. วัตถุดิบต่ำกว่า 20% ของ initQty → ขึ้นสถานะแดง + แจ้งเตือนบนแดชบอร์ด
4. เช็คอิน: เลือกชื่อ → เปิดกล้องถ่ายหน้า → บันทึกรูป+เวลา
   ถ้ากล้องเปิดไม่ได้ (เช่น เปิดแบบ file:// กล้องถูกบล็อก) ให้เช็คอินแบบไม่มีรูปได้ พร้อมข้อความ
   "กล้องจะใช้ได้เมื่อระบบขึ้นเว็บจริง (https)" — ห้ามให้แอพพัง

## ภาค 2 — ออเดอร์จากหน้าลูกค้า (order.html → เมนูออเดอร์เข้า)
- order.html ใช้เมนู/ราคาชุดเดียวกับ MENU และเขียนออเดอร์ลง localStorage "beansERP" เดียวกัน
  (ใช้งานจริงแบบเครื่องเดียว: เมนูเล็ตหน้าร้านให้ลูกค้ากด + หลังบ้านเปิดอีกเมนู/หน้าต่างบนเครื่องเดียวกัน)
- ลูกค้าเลือกเมนู+จำนวน → กรอกชื่อ + โต๊ะ (หรือกลับบ้าน) + เบอร์ (ไม่บังคับ) → กด "สั่งเลย" → order สถานะ "new"
- เมนู 🛎 ออเดอร์เข้า ในหลังบ้าน ต้องเห็นออเดอร์ใหม่เองภายใน ~3 วินาที (ใช้ storage event หรือ refresh อัตโนมัติ)
- ปุ่มเลื่อนสถานะทีละขั้น: ใหม่ → กำลังทำ → เสร็จรอรับ → รับแล้ว+เก็บเงิน
- **กติกาเงิน/สต๊อกยังข้อเดียวเดิม:** ตัดสต๊อกตาม BOM และนับยอดขาย ณ จุด "รับแล้ว+เก็บเงิน" เท่านั้น (= จุดชำระ เหมือนเช็คบิล POS)
- header หลังบ้านมี badge จำนวนออเดอร์ค้าง (new+cooking) เห็นทุกเมนู

### หน้าตา order.html (สำคัญ — ลูกค้าเห็นหน้านี้ ต้องเหมือนแอพสั่งกาแฟจริง ไม่ใช่หน้าแอดมิน)
ออกแบบ mobile-first เรียงจากบนลงล่าง:
1. **Hero ด้านบน**: พื้นน้ำตาลกาแฟเข้ม → ชื่อร้าน "บีนส์ ☕" ตัวใหญ่ + บรรทัดรอง "สั่งเลย รับที่เคาน์เตอร์"
   (ไม่มีเมนูนำทาง ไม่มีปุ่มไปหลังบ้าน)
2. **เมนูเป็นการ์ดใหญ่ 2 คอลัมน์** (มือถือ): อิโมจิเครื่องดื่มตัวใหญ่แทนรูป (☕🥛🍵🍫🍰) บนพื้นครีม
   + ชื่อเมนู + ราคา + ปุ่ม − จำนวน + ทั้งการ์ดกดเพิ่มได้ · การ์ดที่เลือกแล้วมีขอบน้ำตาล+จำนวนมุมการ์ด
3. **แถบตะกร้าลอยติดขอบล่าง (sticky)**: "X รายการ · ยอดรวม Y บาท" + ปุ่มใหญ่ "สั่งเลย →"
   โผล่เฉพาะเมื่อมีของในตะกร้า
4. **กดสั่งเลย → ชั้นยืนยัน (bottom sheet/หน้าย่อย)**: สรุปรายการแก้จำนวนได้ · ช่องชื่อลูกค้า ·
   ปุ่มเลือกโต๊ะ 1-8 เป็นเม็ดกลม + ปุ่ม "กลับบ้าน" · เบอร์โทร (ไม่บังคับ) · ปุ่มยืนยันสั่ง
5. **หน้าขอบคุณ**: เลขคิวตัวใหญ่มาก (เช่น Q07) + ชื่อลูกค้า + รายการย่อ + "รอเรียกที่เคาน์เตอร์ได้เลยค่ะ ☕"
   + ปุ่ม "สั่งเพิ่ม" กลับหน้าเมนู
- หน้านี้ไม่ใช้สกินหลังบ้าน — ใช้โทนแบรนด์ร้าน (น้ำตาลกาแฟ/ครีม อบอุ่น) รูปใหญ่ ตัวหนังสือน้อย ปุ่มใหญ่กดนิ้วโป้งง่าย (หลักคิด: หลังบ้านตามระบบ หน้าบ้านตามแบรนด์)
- ห้ามมีเด็ดขาด: ยอดขายรวมของร้าน สต๊อก ปุ่มแก้ไข/ลบของหลังบ้าน หรือลิงก์ไป app.html

## หน้าจอแต่ละเมนู (ย่อ)
- **แดชบอร์ด**: การ์ด ยอดขายวันนี้ · จำนวนบิล · โต๊ะที่ค้างบิล · เมนูขายดี 3 อันดับ (7 วัน)
  · รายการวัตถุดิบสถานะเหลือง/แดง · ใครเข้างานวันนี้ (พร้อมรูปถ้ามี) · แถบเตือนเรื่องด่วนบนสุด
- **ขาย**: ปุ่มเมนูการ์ดใหญ่กดเร็ว · เลือกโต๊ะ 1-8 หรือกลับบ้าน · ตะกร้า+ยอดรวมสด
  · บันทึกออเดอร์ (โต๊ะค้างบิลรวมยอดได้) · เช็คบิล เงินสด/โอน · ประวัติบิลวันนี้
- **สต๊อก**: ตารางวัตถุดิบ เหลือ/ตั้งต้น + แถบสีเขียว-เหลือง-แดง · ปุ่มรับของเข้า (กรอกจำนวนเติม) · CSV
- **สูตร BOM**: ตารางเมนู×วัตถุดิบ อ่านง่าย ("ลาเต้ 1 แก้ว ใช้อะไรบ้าง") + ต้นทุนวัตถุดิบโดยประมาณต่อแก้วถ้าคำนวณได้
- **เช็คอิน**: ปุ่มชื่อพนักงาน 3 คน → กล้อง → ถ่าย → บันทึกเข้า/ออกงาน · ประวัติวันนี้พร้อมรูปเล็ก
